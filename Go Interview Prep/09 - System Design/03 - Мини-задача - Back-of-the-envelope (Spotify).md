## Мини-задача: Back-of-the-envelope — Spotify

> **Цель:** Отработать расчет DAU → RPS → Read/Write split
> **Алгоритм:** [[01 - Гайд по System Design#2. Оценки (Back-of-the-envelope)]]

---

## Условие

- 50M зарегистрированных пользователей, глобально
- Слушают музыку, лайкают треки, создают плейлисты, ищут

---

## Решение

### DAU
```
Spotify — повседневный продукт, высокая вовлеченность.
40% (реальный ~60%, но 40% — защищаемый ответ)

DAU = 50M × 40% = 20M
```

### Действия за сессию

| Действие | Тип | Кол-во |
|----------|-----|--------|
| Открыть плейлист | Read | 1 |
| Запуск треков | Stream | 5–8 |
| Скипы | Read | 10–15 |
| Лайкнуть трек | Write | 2–3 |
| Добавить в плейлист | Write | 4–5 |

Итого: ~30 запросов/день

### RPS

```
Avg: 20M × 30 / 86400 ≈ 6,900 RPS
Peak (дроп альбома, DAU +50%): 25M × 40 / 86400 ≈ 11,500 RPS
Peak = 1.67× avg
```

### Read / Write

```
Read:  ~95% (слушать, искать)
Write:  ~5% (лайки, плейлисты)
→ Read-heavy: кэш + CDN
```

---

## Итог

| Метрика | Значение |
|---------|----------|
| DAU | 20M → 25M (пик) |
| Avg RPS | ~6,900 |
| Peak RPS | ~11,500 |
| Read/Write | 95% / 5% |

---

## Инсайт: два типа трафика

```
Metadata API (REST):  likes, playlists, search → обычное кэширование
Streaming API:        аудио → chunks по HLS/DASH → CDN
                      НЕ UDP — нет контроля качества
```
