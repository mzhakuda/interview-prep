## Условие

Спроектировать Load Balancer для высоконагруженного сервиса.

**Требования:**
- Распределять входящий трафик между несколькими серверами
- Определять когда сервер недоступен и убирать его из rotation
- Поддерживать 1M+ RPS
- Не быть единственной точкой отказа сам по себе

## Что нужно разобрать

### 1. Алгоритмы балансировки

| Алгоритм | Как работает | Когда использовать |
|----------|-------------|-------------------|
| Round Robin | По очереди | Одинаковые серверы, легкие запросы |
| Weighted Round Robin | По очереди с весом | Серверы разной мощности |
| Least Connections | К серверу с наименьшим кол-вом активных соединений | Запросы разной длины |
| IP Hash | hash(client_ip) → всегда один сервер | Нужны sticky sessions |
| Random | Случайный сервер | Простые случаи |

### 2. Health Checks

Как LB понимает что сервер живой:
```
Active:  LB сам шлет запросы → GET /health каждые N сек
Passive: LB смотрит на ответы реальных запросов → 5xx подряд = unhealthy
```

### 3. Sticky Sessions

Проблема: пользователь залогинился на Server 1, следующий запрос попал на Server 2 → сессия потеряна.

Решения:
```
Cookie-based: LB ставит cookie с ID сервера
IP Hash:      hash(IP) → всегда один сервер
              Минус: если юзер за NAT — все идут на один сервер
Лучше всего:  Stateless сервисы + сессии в Redis (не нужны sticky)
```

### 4. L4 vs L7

```
L4 (Transport):
  Балансирует по IP + TCP/UDP порту
  Быстрее — не читает содержимое пакетов
  Не понимает HTTP

L7 (Application):
  Балансирует по HTTP заголовкам, URL, cookies
  Может роутить /api/* → API серверы, /static/* → CDN
  Медленнее, но умнее
```

### 5. LB как SPOF — главная проблема

```
Проблема: LB сам упал → весь трафик упал 💥

Решение: Active-Passive пара
  LB Primary + LB Standby
  Виртуальный IP (VIP) — один адрес на оба
  Keepalived следит: Primary упал → Standby берет VIP

Или: DNS Round Robin
  DNS отдает несколько IP → клиент сам выбирает
  Минус: DNS кэш → failover медленный (TTL)
```

### 6. Глобальная балансировка (GeoDNS)

```
Запрос из Казахстана → DNS → IP ближайшего региона
Запрос из США       → DNS → IP американского региона

Инструменты: AWS Route53, Cloudflare
```
